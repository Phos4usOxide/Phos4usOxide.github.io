Images

Images show information far more effectively than text and are easier to remember than text. A common phrase is "a picture is worth a thousand words", so whenever applicable, it's good to provide images to help represent and show what's being talked about, so users have something to visualize. Images also increase the site's user retention, aesthetics, and professionalism. Let's check out some guidelines for image usage!

Quality

Images should have a fitting resolution for their use. They shouldn't be too big or too small unless the context specifically requires it. To increase quality, ensure the images have a high enough bit depth, good contrast, are clear instead of blurry, and the focus is obvious.

Content

Images should be relevant to help the users understand and visualize what's being talked about. The images on this page aren't too relevant, but they follow the rest of the guidelines. Other types of images would work better in certain cases, like charts and infographics. Lastly, limit the use of text in images (when possible) since it defeats the image's purpose.

Accessibility

If an image fails to load, it should have alternate text so the user knows what is supposed to be shown. This alternate text should be descriptive enough so the meaning is still understood and the user can picture it themself. Ensure images are correctly responsive so they can can be viewed correctly on different sized devices and app windows.

General

Images should not be over-used or under-used and should only be used in places that make sense so the balance is good. Ensure the image credit is clearly shown if applicable like with the examples on this page. These particular images are actually free to use and the credit isn't even needed. Lastly, images must not take a while to load so smaller file sizes are better. For example, JPG is significantly faster to load than PNG since the file size is often 8-10 times smaller.
